= Crie um relatório para visualizar VMs de armazenamento paradas
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Você pode criar um relatório para exibir uma lista de todos os SVMs interrompidos.

.Antes de começar
* Você deve ter a função de Administrador de Aplicativos ou Administrador de Armazenamento.


Use as etapas a seguir para criar uma exibição personalizada que exiba VMs de armazenamento interrompidas e, em seguida, agende um relatório para ser gerado para essa exibição.

.Passos
. No painel de navegação esquerdo, clique em *Armazenamento* > *VMs de armazenamento*.
. No menu Exibir, selecione *Saúde* > *Todas as VMs de armazenamento*.
. Selecione *Mostrar/Ocultar* para remover quaisquer colunas que você não deseja no relatório.
. Arraste a coluna "`Estado`" para perto da coluna "VM de armazenamento`".
. Clique no ícone de filtro, adicione o seguinte filtro e clique em *Aplicar filtro*:
+
** O estado está parado


. Salve a visualização com um nome específico que reflita o que a visualização está mostrando, por exemplo "`SVMs interrompidos`" e clique na marca de seleção (image:../media/blue_check.gif[""] ).
. Clique no botão *Relatórios agendados* na página de inventário.
. Clique em *Adicionar agendamento* para adicionar uma nova linha à página *Agendamentos de relatórios* para que você possa definir as características do agendamento para o novo relatório.
. Digite um nome para o agendamento do relatório e preencha os outros campos do relatório e clique na marca de seleção (image:../media/blue_check.gif[""] ) no final da linha.
+
O relatório é enviado imediatamente como um teste.  Depois disso, o relatório é gerado e enviado por e-mail aos destinatários listados usando a frequência especificada.



Com base nos resultados mostrados no relatório, talvez você queira investigar por que o SVM foi interrompido para ver se deve reiniciá-los.
